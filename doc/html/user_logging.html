Logging

Introduction

Logging can be enabled in SNPE during the execution of a network. It is highly recommended to enable logging once at the beginning of the program to capture logs from all processes. Enabling logging may have a performance impact. The following sections demonstrate the logging support in SNPE:

  1. C++ Logging
    1. C++ Log Levels
    2. C++ Logging APIs
  2. Java Logging
    1. Java Log Levels
    2. Java Logging APIs

C++ Logging

The section demonstrates the available SNPE log levels and logging APIs for C++. The following header needs to be included to use the logging functionality.

#include <SNPE/SNPEFactory.hpp>

For a complete example, please refer to the sample application file located at $SNPE_ROOT/examples/NativeCpp/SampleCode/jni/main.cpp.

C++ Log Levels

The following are the different log levels supported by SNPE logging APIs in the increasing order of importance.

Level SNPE C++ Log Level
VERBOSE zdl::DlSystem::LogLevel_t::LOG_VERBOSE
INFO zdl::DlSystem::LogLevel_t::LOG_INFO
WARN zdl::DlSystem::LogLevel_t::LOG_WARN
ERROR zdl::DlSystem::LogLevel_t::LOG_ERROR
FATAL zdl::DlSystem::LogLevel_t::LOG_FATAL

When a specific log level is set, logs with that importance level and higher importance levels will be logged.

C++ Logging APIs

SNPE provides different variants of logging initialization for Android platforms and non-Android platforms. Please see zdl::SNPE::SNPEFactory::initializeLogging() for more information.

On Android platforms, the SNPE logs are printed to android logcat. SNPE logging can be initialized with a specific log level by, 

    zdl::SNPE::SNPEFactory::initializeLogging(zdl::DlSystem::LogLevel_t logLevel);

On non-Android platforms, the SNPE logs are printed to console output and into log files created in the specified log path. SNPE logging can be initialized with a specific log level and a log path by, 

    zdl::SNPE::SNPEFactory::initializeLogging(zdl::DlSystem::LogLevel_t logLevel, std::string logPath);

The SNPE logging level can be changed during execution to a specific log level by, 

    zdl::SNPE::SNPEFactory::setLogLevel(zdl::DlSystem::LogLevel_t logLevel);

The SNPE logging can be terminated by, 

    zdl::SNPE::SNPEFactory::terminateLogging();

Java Logging

The section demonstrates the available SNPE log levels and logging APIs for Java. The following packages needs to be imported to use the logging functionality.

import com.qualcomm.qti.snpe.SNPE;
import com.qualcomm.qti.snpe.NeuralNetwork;

For a complete example, please refer to the sample android application files located at $SNPE_ROOT/examples/android/samples/image-classifiers/app/src/.

Java Log Levels

The following are the different log levels supported by SNPE logging APIs in the increasing order of importance.

Level SNPE Java Log Level
VERBOSE NeuralNetwork.LogLevel.LOG_VERBOSE
INFO NeuralNetwork.LogLevel.LOG_INFO
WARN NeuralNetwork.LogLevel.LOG_WARN
ERROR NeuralNetwork.LogLevel.LOG_ERROR
FATAL NeuralNetwork.LogLevel.LOG_FATAL

When a specific log level is set, logs with that importance level and higher importance levels will be logged.

Java Logging APIs

On Android platforms, the SNPE logs are printed to android logcat. SNPE logging can be initialized with a specific log level by, 

    SNPE.logger.initializeLogging(android.app.Application application, NeuralNetwork.LogLevel logLevel);

The SNPE logging level can be changed during execution to a specific log level by, 

    SNPE.logger.setLogLevel(NeuralNetwork.LogLevel logLevel);

The SNPE logging can be terminated by, 

    SNPE.logger.terminateLogging();
